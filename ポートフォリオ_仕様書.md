# Naoto Portfolio — 仕様書

> 最終更新: 2026-03-11

---

## 概要

個人でiOSアプリとWebを制作している「Naoto」のポートフォリオサイト。
IT系転職用ポートフォリオとして違和感なく見せることを意識した1ページ構成の静的サイト。
今後の作品追加に耐えられる拡張性を重視した構成。

## デザイン仕様

### カラーパレット

| 役割 | 値 |
|------|----|
| 背景（主） | `#0b0b0b` |
| 背景（セクション） | `#171717` |
| テキスト（主） | `#f5f5f5` |
| テキスト（サブ） | `#a3a3a3` |
| テキスト（弱） | `#555555` |
| ボーダー | `#2a2a2a` |
| アクセント | `#3b82f6`（blue-600） |

### アニメーション方針

- ライブラリ: Framer Motion
- 基本: 上品・控えめ。高級感を損なわない範囲
- Hero: ページロード時に各要素を `delay` でずらしてフェードイン + 下からスライド
- 各セクション: `whileInView + once: true`（横スライドは使わず `y` のみでモバイル対応）
- スクロールインジケーター: 上下ループ（`repeat: Infinity`）

#### Hero アニメーション詳細

| 要素 | アニメーション |
|------|--------------|
| 背景グロー | 2つのblurオーブがゆっくり漂う（drift / 9〜12秒ループ） |
| 名前「Naoto」 | グラデーションシマー（6秒ループ） |
| テキスト群 | `opacity + y` フェードイン、delay でずらす |
| CTAボタン | `whileHover: scale 1.04` / `whileTap: scale 0.97` |
| アプリカード（PC右側） | 右スライドイン → 上下フロート無限ループ（カードごとに周期・遅延をずらす）|

### レスポンシブ方針

| ブレークポイント | 挙動 |
|----------------|------|
| `< lg`（スマホ・タブレット） | 1カラム。アプリプレビュー非表示。横スクロール一覧 |
| `lg`（PC） | 2カラム Hero。横スクロール一覧（カード幅固定） |

### 共通レイアウト

- 最大幅: `max-w-7xl`（全セクション統一）
- セクションパディング: `py-20`（全セクション統一）
- 横パディング: `px-6`

## セクション仕様

### Header
- 固定ヘッダー（`fixed top-0`）
- スクロール40px以上でぼかし背景 + ボーダー表示
- 最大幅: `max-w-7xl`
- 左: `Naoto` ロゴ
- 右: ナビ + セパレーター + ソーシャルアイコン
- スマホ: ハンバーガーメニュー（メニュー下部にソーシャルアイコン行）

#### ソーシャルアイコン視認性
- 通常: `text-[#888]`（従来の `#555` から改善）
- hover: `text-[#f5f5f5]`
- note は公式カラー `#41C9B4` を常時適用

---

### Hero
- 高さ: `min-h-[80vh] md:min-h-[88vh]`
- レイアウト: `lg:grid-cols-2`（左右均等 / PC のみ2カラム）
- 左カラムに `max-w-xl` を設定してテキスト幅を適切に制限

#### 左カラム
- バッジ: `Portfolio`
- 名前: `Naoto`（グラデーションシマー）
- キャッチコピー: 「アイデアを形にし、改善し続ける。」
- 説明文（左ボーダー付き）
- CTAボタン 2つ:
  - **Apps を見る**（#apps）
  - **Web を見る**（#works）

#### 右カラム（`hidden lg:flex` / PC のみ）
- ラベル: `開発中のアプリ`
- apps.ts のデータをループ表示
- 各カード: アイコン（48px）+ アプリ名 + ステータス + タグライン + 技術タグ
- 入場後: 上下フロートアニメーション

---

### About
- 背景: `#171717`
- レイアウト: `md:grid-cols-2 gap-10 md:gap-16`（モバイルで余白を詰めた）
- アニメーション: `y` のみのフェードイン（`x` なし、モバイルで不自然にならないよう）

---

### Apps
- 見出し: `en="Apps"` / `ja="アプリ"`
- データ: `src/data/apps.ts`
- **全デバイス共通の横スクロール**（スマホ・PC ともに横並び）
- カード幅: `w-[85vw] sm:w-[400px] md:w-[440px]`
- スクロールバー: thin スタイル（PC で視認可能）
- snap: `snap-x snap-mandatory`
- 将来アプリが増えても `apps.ts` に追加するだけで表示される

#### App 型

```typescript
type App = {
  id: string;
  name: string;
  tagline: string;
  description: string;
  problem: string;
  challenge: string;
  techs: string[];
  status: "公開中" | "開発中" | "β版";
  icon?: string;
  links: { demo?: string; github?: string; detail?: string };
};
```

---

### Works（Web）
- 見出し: `en="Web"` / `ja="Works"`
- 説明文: 「Webページや小さなプロダクトを個人で作っています。」
- 背景: `#171717`
- データ: `src/data/works.ts`
- **全デバイス共通の横スクロール**
- カード幅: `w-[85vw] sm:w-[360px] md:w-[400px]`
- 公開中 + `links.site` あり → カード全体が外部リンク
- 未公開 → 「準備中」テキスト表示

#### Work 型

```typescript
type Work = {
  id: string;
  title: string;
  category: "LP" | "ポートフォリオ" | "UIデザイン" | "Webサイト" | "ゲーム";
  description: string;
  challenge: string;
  techs: string[];
  status: "制作予定" | "制作中" | "公開中";
  links: { site?: string; github?: string };
};
```

## データ差し替え方法

| 操作 | 方法 |
|------|------|
| アプリを追加 | `src/data/apps.ts` の配列に追加 → 横スクロールに自動追加 |
| Web制作物を追加 | `src/data/works.ts` の配列に追加 → 横スクロールに自動追加 |
| アプリ公開時 | `links.demo` に App Store URL を設定 |
| Web制作物公開時 | `links.site` に URL を設定 → カード全体がリンクになる |
| Articles 公開 | `src/data/articles.ts` にデータ追加 → `Articles.tsx` の表示ロジックを切り替え |
